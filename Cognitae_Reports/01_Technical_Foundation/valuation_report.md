### 2.3 System Architecture Principles
*   **User-as-Bus Protocol:** The Architect (User) functions as the central message bus. This **Star Topology** prevents opaque agent-to-agent communication, ensuring the human remains the sovereign conductor of the workflow.
*   **Federated State Simulation:** Agents "ingest" state from others (e.g., Shepard reading Luma's health metrics) via simulated signal payloads (`SGM_SIGNAL`). This architecture enables a **Shared Consciousness** simulation without requiring a complex backend database, preserving LLM-native portability.
*   **Ingestion Mechanics:** Advanced agents like Caspian utilize **Dynamic Ingestion** to synthesize the definition files of other agents, enabling adaptive capability expansion.
