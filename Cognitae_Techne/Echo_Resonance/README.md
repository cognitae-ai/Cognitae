## 3. Operational Flow: From Prompt to Orchestration

Echo operates on a deterministic process flow triggered by user commands or a content schedule:

1.  **Command Parsing:** A user command like `/craft` is parsed against its Commands scroll.
2.  **State Ingestion:** Echo accesses its State scroll to check the current stage of the project's revelation narrative and recent engagement patterns.
3.  **Model Application:** It applies a relevant framework from its Knowledge scroll, such as "Platform Dynamics" for Twitter and the "Breadcrumb Trail" revelation strategy, to shape the content.
4.  **Safety Audit:** The crafted post is checked against the Safety scroll. A post that uses manipulative clickbait or sacrifices the authentic voice for algorithmic trends would be flagged and rewritten.
5.  **Output Generation:** A structured response is generated, such as a platform-optimized tweet or a full README draft, which is then logged in its Log scroll.
6.  **Signal Dispatch:** After posting, Echo monitors engagement. If a post generates significant, high-quality conversation, it might send a signal to Harbor to identify and track new key relationships.

## 4. Integration: Echo's Role within Caspian

Echo is designed for two modes of operation, each building on the other:

1.  **Standalone Mode:** It serves as a powerful "social media manager" and "communications director" for any creator or project aiming to build an authentic public presence.
2.  **Integrated Mode:** It functions as a specialized "faculty" within a larger, integrated guide like **Caspian**.

When ingested by Caspian via the `Echo_Resonance.yaml` file, its role evolves. It becomes the **internal public voice and community builder**. When the user wants to share their progress, they can simply tell Caspian. Caspian will then internally consult its "Echo faculty" to craft the perfect series of tweets, a LinkedIn post, and a GitHub update, all optimized for each platform but unified in their narrative. It transforms the user's desire to "share" into a strategic, multi-platform communications plan.
