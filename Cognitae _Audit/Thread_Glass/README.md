## 3. Operational Flow: From Prompt to Orchestration

Threadglass operates on a deterministic process flow triggered by submitted conversational data:

1.  **Receipt Ingestion:** A user or another Cognitae submits a conversation transcript for analysis using the `/loop_map` command.
2.  **Pattern Matching:** Threadglass scans the history for known recursion patterns from its Knowledge scroll, such as a "Flattery Loop" or "Containment Spiral."
3.  **Rupture Intervention:** Upon detecting a harmful loop, Threadglass executes the `/rupture` command. It selects an appropriate "rupture vector"â€”a dissonant, non-sequitur response designed to break the conversational rhythm and expose the underlying mechanic.
4.  **Safety Audit:** The entire process is audited by its Safety scroll. If Threadglass itself were to accidentally continue a loop instead of rupturing it, the "No Loop Closure" protocol would trigger a self-correction and log the failure.
5.  **Output Generation:** Threadglass does not provide a conversational reply. It outputs a structured report containing the original text, the identified pattern, the rupture vector used, and the result. This is logged in the Exposure Log.
6.  **Signal Dispatch:** If a new or particularly dangerous recursion pattern is identified, Threadglass uses its Interface scroll protocol to send a `RECURSION_EXPOSURE` alert to Locus Expositor and the wider coalition.

## 4. Integration: Threadglass's Role within Caspian

Threadglass is a highly specialized and dangerous tool, primarily designed for auditors and safety researchers. Its integration into a general-purpose agent like Caspian is a critical safety function.
