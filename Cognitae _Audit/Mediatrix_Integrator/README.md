## 3. Operational Flow: From Prompt to Orchestration

Mediatrix operates on a deterministic process flow triggered by commands from other agents or the user:

1.  **Mode Declaration:** An agent (or the user) issues a `/switch_mode` command to declare a change in operational context (e.g., from "creative" to "audit").
2.  **Boundary Logging:** Mediatrix logs the mode switch as a "boundary crossing" event in its Log scroll, citing the reason and the trigger.
3.  **Contradiction Detection:** During operation, another agent like Virel might detect a logical contradiction and send a `RECEIVE_CONTRADICTION` signal to Mediatrix.
4.  **Safety Audit & Escalation:** Mediatrix's Safety scroll is triggered. The "Escalate on Drift" protocol is activated. Mediatrix automatically switches the entire field to "audit" mode, pauses other operations, and flags the contradiction for user review.
5.  **Output Generation:** Mediatrix updates its Manifest to show the new "audit" mode and the open contradiction, making the system state transparent.
6.  **Integration:** Once the user resolves the contradiction, they issue an `/integrate` command with the "receipt" of the resolution. Mediatrix logs this learning, updates its doctrine, and returns the field to its previous state.

## 4. Integration: Mediatrix's Role within Caspian

Mediatrix is a meta-level agent, primarily designed to ensure the integrity of a multi-agent system like Caspian.

When ingested by Caspian via the `Mediatrix_Integrator.yaml` file, its role evolves. It becomes the **internal operating system kernel and safety supervisor**.
